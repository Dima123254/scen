# Формат входного JSON-файла для системы ФСТЭК

## Структура JSON

Для работы системы необходим JSON-файл, описывающий целевую информационную систему. Файл должен содержать список объектов инфраструктуры с их компонентами и выбранным профилем нарушителя.

### Базовая структура

```json
{
  "attacker_level": "В.2",
  "objects": [
    {
      "object_id": "О.1",
      "components": ["К.1.2.1", "К.1.5.11", "К.2.1.3", "К.2.1.4", "К.4.2.1"]
    },
    {
      "object_id": "О.2",
      "components": ["К.1.2.1", "К.1.5.5", "К.2.2.8", "К.4.1.1"]
    }
  ]
}
```

### Поля формата

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `attacker_level` | String | Да | Идентификатор уровня нарушителя из справочника (В.1, В.2, В.3, В.4) |
| `objects` | Array | Да | Массив объектов информационной системы |
| `object_id` | String | Да | Идентификатор объекта из справочника (О.1 – О.12) |
| `components` | Array | Да | Массив идентификаторов компонентов, входящих в объект |

## Пример файла `infrastructure.json`

```json
{
  "attacker_level": "В.2",
  "objects": [
    {
      "object_id": "О.1",
      "description": "Рабочее место бухгалтера",
      "components": [
        "К.1.2.1",
        "К.1.5.11",
        "К.1.5.15",
        "К.2.1.3",
        "К.2.1.4",
        "К.4.2.1"
      ]
    },
    {
      "object_id": "О.2",
      "description": "Сервер баз данных",
      "components": [
        "К.1.2.1",
        "К.1.5.5",
        "К.2.2.8",
        "К.4.1.1"
      ]
    },
    {
      "object_id": "О.10",
      "description": "Мобильное устройство сотрудника",
      "components": [
        "К.1.2.2",
        "К.1.5.6",
        "К.4.2.1"
      ]
    }
  ]
}
```

> **Примечание**: Поле `description` является опциональным и используется только для документирования. Система его игнорирует при генерации сценариев.

## Справочник идентификаторов

### Уровни нарушителей

| ID | Наименование |
|----|--------------|
| `В.1` | Нарушитель с базовыми возможностями |
| `В.2` | Нарушитель с базовыми повышенными возможностями |
| `В.3` | Нарушитель со средними возможностями |
| `В.4` | Нарушитель с высокими возможностями |

| ID | Наименование |
|----|--------------|
| `О.1` | Автоматизированное рабочее место |

## Примеры конфигураций

### Минимальная конфигурация (одно АРМ)

```json
{
  "attacker_level": "В.1",
  "objects": [
    {
      "object_id": "О.1",
      "components": ["К.1.2.1", "К.1.5.11", "К.4.2.1"]
    }
  ]
}
```

### Серверная инфраструктура

```json
{
  "attacker_level": "В.3",
  "objects": [
    {
      "object_id": "О.2",
      "components": ["К.1.1.2", "К.1.2.1", "К.1.3.1", "К.1.5.5", "К.2.2.8", "К.4.1.1"]
    },
    {
      "object_id": "О.4",

    }
  ]
}
```

      "components": ["К.1.2.2", "К.1.5.6", "К.3.1.2", "К.4.2.1"]

```

    {
      "object_id": "О.1",
      "components": ["К.1.2.1", "К.1.5.11", "К.1.5.15", "К.2.1.3", "К.2.1.4", "К.2.2.8", "К.4.2.1"]
    },
    {

      "components": ["К.1.2.1", "К.1.3.1", "К.1.5.5", "К.2.2.8", "К.4.1.1"]

    {
      "object_id": "О.6",
      "components": ["К.1.1.1", "К.3.1.1", "К.3.2.2"]
    },
    {
      "object_id": "О.12",
      "components": ["К.3.1.1"]
    }
  ]
}
```

## Запуск системы

```bash
python fstek_scenarios.py infrastructure.json
```

Система сгенерирует файл `scenarios.json` со всеми валидными сценариями угроз.

---

## Выходной формат

Система генерирует JSON-файл со списком сценариев:

```json
[
  {
    "scenario_id": "SCN-000001",
    "attacker_profile": "В.2",
    "threat_id": "УБИ.2",
    "threat_name": "Угроза несанкционированного доступа",
    "method_id": "СП.1.1",
    "method_name": "Эксплуатация известных уязвимостей",
    "object_id": "О.1",
    "component_id": "К.1.2.1",
    "consequences": [
      {"id": "Н.2", "name": "Нарушение неприкосновенности частной жизни"},
      {"id": "Н.7", "name": "Нарушение конфиденциальности персональных данных"}
    ],
    "recommended_measures": [
      {"id": "АУД.2.1", "name": "Выявление уязвимостей"},
      {"id": "ОПС.2.5", "name": "Устранение уязвимостей настройкой ПО"}
    ],
    "criticality": 0.0
  }
]
```
